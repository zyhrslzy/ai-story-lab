# 项目状态

## Day 3 目标

提高 AI 故事生成可靠性：把模型原始输出经过确定性的解析与校验链路，区分可恢复错误，最多重试一次，并在最终失败时提供可配置、可识别的 Mock 降级或安全失败体验。

## 实际完成内容

- 建立“模型原始输出 → JSON 解析 → Zod 结构校验 → 独立故事图规则校验”的服务端链路。
- 故事图规则覆盖起点、节点与选择 ID、引用、结局数量、可达性、孤立节点、循环和两次选择路径。
- 细分超时、网络、模型、JSON、Schema、故事图和未知错误；前端只接收稳定错误码、安全中文提示和请求 ID。
- 对可恢复错误最多自动重试一次，并把两次尝试限制在 15 秒总等待预算内。
- 增加可配置 Mock 降级，并把来源、是否降级、尝试次数和降级原因随故事会话保存。
- 生成失败时保留表单内容、显示“重新生成”，并使用同步锁防止重复提交形成并发请求。
- 增加最小结构化生成记录，包含 requestId、生成器、Prompt 版本、时间、耗时、成功、重试、降级和最终错误类型，不记录完整输入、Prompt 或模型输出。

## 测试结果

- `npm run lint`：通过。
- `npm run typecheck`：通过。
- `npm run test`：6 个测试文件、37 个测试全部通过。
- 自动化测试覆盖合法故事、非 JSON、缺失字段、重复节点 ID、失效引用、零或单一结局、结局包含选择、循环、孤立节点、路径长度、超时重试、重试后降级或失败、测试环境阻止真实模型、表单保留和并发提交锁。

## 手动验收结果

- **成功流程**：在 390 × 844 手机视口中使用 Mock，从填写四项设定、两次选择到希望结局完整走通；刷新后仍恢复在同一结局。
- **失败后重试并降级**：使用本地假模型连续返回非法 JSON，确认系统只重试一次，结构化记录为 `retried=true`、`fallbackUsed=true`、`finalErrorType=INVALID_JSON`，游玩页显示“AI 暂时不可用，已为你加载示例故事。”。
- **最终失败**：关闭 Mock 降级并重复同一故障，确认 API 返回安全提示，四项表单内容保留，按钮变为“重新生成”；结构化记录为失败、已重试、未降级。
- 以上验收均未使用真实 API Key，也未发送真实 OpenAI 请求。

## 遗留问题

- 尚未用真实 OpenAI Key 采集延迟、重试恢复率和输出错误分布。
- 当前结构化日志只输出到服务端控制台，尚未接入第三方监控或分析平台。
- Mock 内容不能用于评价 AI 生成质量；质量样本必须按来源和 Prompt 版本过滤。
